## concurrent_map 测试结果
主机配置：4C/6G

```sh
worker线程数     读写锁数   耗时(s)real/user/sys
4                   1      2.663/1.426/2.094
4                   2      1.760/1.290/1.498
4                   3      1.207/1.072/0.985
4                   4      1.035/0.895/0.709
4                   5      0.921/0.916/0.700
```
结论worker线程数保持与主机核数一致，读写锁数量值越大，并发读写的耗时越低，但越耗资源。
